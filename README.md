1. タイトル（仮称）
『SciVis Archive』 ～ 3DCG・画像処理研究の成果物管理ポートフォリオ ～
2. テーマ・コンセプト
【現状の課題】 大学の研究活動（POV-Rayによる3D描画やPython/OpenCVによる画像処理）において、生成された画像データがローカル環境に散在しており、制作の履歴管理や第三者への提示（就職活動時のアピール等）がスムーズに行えない。
【目的・解決策】 Webブラウザ上で動作する「画像管理システム」を構築する。 制作した画像を「タイトル」「解説コメント」「制作日時」と共にデータベースへ保存・蓄積することで、自身のスキル成長を可視化する。また、本システム自体の開発を通じて、Webアプリケーションのバックエンド（PHP/MySQL）技術を習得することを目的とする。
3. 実装機能と構成（解決の手段）
本システムは、PHPとMySQLを用いた動的なWebアプリケーションとして構築する。
【主な機能】
1.	作品投稿機能：
o	画像ファイル（.jpg, .png等）のアップロード機能。
o	作品タイトル、解説テキスト（改行対応）の入力・保存。
o	アップロード日時の自動記録。
2.	作品一覧表示機能：
o	投稿された画像をサムネイルとして一覧表示。
o	CSSグリッドレイアウトを用い、視認性の高いタイル状のデザインを採用。
3.	管理機能（削除）：
o	不要な投稿や古いデータを削除する機能。
o	データベース上のレコード削除と同時に、サーバー上の画像ファイルも自動削除する仕組みを実装し、サーバー容量を圧迫しない設計とする。
【技術要件とスコア（合計：8点）】
•	[4点] 画像アップロード機能： input type="file" および move_uploaded_file関数を使用。
•	[2点] CSSによる装飾： grid-template-columns 等を使用し、レスポンシブなカード型レイアウトを実装。
•	[2点] 改行を含む文章投稿： textareaタグを使用し、nl2br関数等で改行を表示に反映。
4. 画面遷移・操作フロー（使い方）
1.	メイン画面（一覧兼投稿画面）：
o	画面上部のフォームから、作品タイトル・コメント・画像ファイルを選択し「アップロード」ボタンを押下する。
2.	システム処理：
o	画像はサーバー内の images/ ディレクトリに、固有のファイル名（日時情報を付与）で保存される。
o	画像のパスとメタデータがMySQLデータベース（tb_portfolio）に保存される。
3.	一覧確認：
o	投稿完了後、画面下部のグリッドエリアに新しい作品が即座に追加表示される。
4.	削除処理：
o	削除フォームに対象IDを入力して実行すると、該当作品が削除される。
